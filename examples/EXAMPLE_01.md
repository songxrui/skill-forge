# EXAMPLE_01: skill-forge 实战 — dbs-learning R3 升级全过程

## 输入
```
skill: dbs-learning (STUB/THIN 级别)
当前状态: SKILL.md 仅 456B, 无 description, 无工作流, 无失败兜底
```

## 锻造过程

### Step 1: 读取原始素材
- 读取 `dbs-learning/SKILL.md` (456B)
- 提取: "交互式学习系统。把一个课题拆成连续学习文章，根据反馈调整深度"
- 无 references/ → 以 SKILL.md 全文为原始素材

### Step 2: 提取模式
- **核心定义**: 交互式学习系统 — 拆课题 → 连续文章 → 用户反馈 → 调整深度
- **关键方法论**: ①课题拆分粒度 ②反馈驱动的深度调整 ③连续学习节奏
- **工作流程**: 输入课题 → 拆分 → 写第1篇 → 收集反馈 → 调整写第2篇 → 循环
- **输入**: 学习课题名称 + 当前知识水平
- **输出**: 连续学习文章流，每篇根据上篇反馈调整

### Step 3: 锻造 description
```
description: "交互式学习系统。把一个课题拆成连续学习文章，根据用户在上一篇的真实反馈调整下一篇的深度和方向，形成个性化学习流。触发词：dbs-learning、怎么学、学什么、课题学习、系统学。不适用：快速查概念→exa-search; 读书→ljg-book。正例：'我想系统学习奥地利经济学'→触发; 'dbs-learning 帮我学微积分'→触发。反例：'奥地利经济学是什么'→不触发→exa-search; '推荐一本经济学书'→不触发→weread-skills。"
```

### Step 4-6: 锻造 body + 验证清单 + 失败兜底 + G1-G6
（产出完整 SKILL.md，含 3 Phase 工作流、4 种失败兜底、5 项验证清单、6 项门禁表）
